## 数据结构

### Segment

表示视频中的一个段落（非静音区间）：

```python
@dataclass
class Segment:
    index: int                    # 段落索引
    start_time: float             # 开始时间 (秒)
    end_time: float               # 结束时间 (秒)
    duration: float               # 时长 (秒)

    # 基础评分 (4维 × 25分)
    score_start: float = 0        # 清晰的开始
    score_end: float = 0          # 清晰的结束
    score_fluency: float = 0      # 中间流畅
    score_rhythm: float = 0       # 节奏自然
    total_score: float = 0        # 基础分合计

    # 内部中断信息
    internal_silences: List[Tuple[float, float]]  # 内部静音段
    interruption_count: int = 0                    # 中断次数
    interruption_duration: float = 0               # 中断总时长

    # 转录与流畅度
    transcript: str = ""          # Whisper 转录文本
    repeat_count: int = 0         # 短语重复次数
    stutter_count: int = 0        # 语气词卡顿次数
    is_natural_end: bool = False  # 是否自然结束
    is_interrupted: bool = False  # 是否突然中断

    # 调整分与去重
    adjusted_score: float = 0     # 调整分 (0-100)
    is_duplicate: bool = False    # 是否被标记为重复
    duplicate_with: List[int]     # 与哪些段落重复
```

### ClipInfo

表示单个视频的粗剪结果，用于跨视频去重：

```python
@dataclass
class ClipInfo:
    video_name: str               # 视频文件名 (不含扩展名)
    clip_path: str                # 粗剪输出文件路径
    transcript: str               # 最佳段落的转录文本
    adjusted_score: float         # 最佳段落的调整分
    is_natural_end: bool          # 最佳段落是否自然结束
    duration: float               # 最佳段落时长
    is_cross_duplicate: bool = False  # 是否被跨视频去重标记
    duplicate_of: str = ""            # 与哪个视频重复
```

## 核心模块

### 模块1: 静音检测

```python
def detect_silence(video_path) -> List[Tuple[float, float]]
```

使用 FFmpeg 的 `silencedetect` 滤镜检测视频中的静音段。

**参数**:
- `silence_noise`: 静音阈值 (dB)，默认 -30，越小越严格
- `silence_duration`: 静音最小时长 (秒)，默认 0.8

**注意**: FFmpeg 的静音检测结果输出在 stderr 中。

---

### 模块2: 段落识别

```python
def identify_segments(silences, total_duration) -> List[Segment]
```

根据静音段将视频切分为段落：
1. 第一个段落：视频开始 → 第一个静音开始
2. 中间段落：前一个静音结束 → 下一个静音开始（≥1秒才保留）
3. 最后一个段落：最后一个静音结束 → 视频结束

---

### 模块3: 评分系统

```python
def score_segment(seg, silences, total_duration) -> Segment
```

4维度评分，每维25分，总分100：

**清晰的开始 (25分)**:

| 条件 | 得分 |
|------|------|
| 前方静音 ≥ 1.0秒 | 25 |
| 前方静音 ≥ 0.5秒 | 20 |
| 前方有静音但 < 0.5秒 | 10 |
| 位于视频开头 (< 0.5秒) | 15 |
| 其他 | 5 |

**清晰的结束 (25分)**: 同理，检测后方静音。

**中间流畅 (25分)**:

| 内部中断次数 | 得分 |
|-------------|------|
| 0 | 25 |
| 1-2 | 20 |
| 3-4 | 15 |
| 5+ | max(5, 25 - 中断数 × 3) |

**节奏自然 (25分)**: 由三部分组成：

- 停顿占比 (15分): 占比 < 5% 得15分，逐级递减
- 最大单次停顿 (10分): < 0.8秒得10分，逐级递减
- 极短片段惩罚: < 8秒封顶15分，< 15秒封顶20分

---

### 模块4: 转录

```python
def transcribe_segment(video_path, seg, work_dir) -> str
```

1. 使用 FFmpeg 提取段落音频 (16kHz, mono, WAV)
2. 调用 Whisper CLI 转录为中文文本
3. 若 Whisper 未安装，自动跳过，使用纯音频评分

---

### 模块5: 流畅度分析

```python
def analyze_fluency(transcript) -> Tuple[int, int, bool, bool]
```

返回 `(repeat_count, stutter_count, is_natural_end, is_interrupted)`。

**重复检测**: 滑动窗口算法，检测 2-4 字短语在后续 10 字窗口内是否重复出现。

**卡顿检测**: 匹配以下模式：
- 单字语气词: `[嗯啊呃]`
- 口癖短语: `那个`、`就是说`
- 省略号: `...`、`…`

**突然中断检测**: 检测文本是否以以下20种连接词/未完成标记结尾：

```
的时候 | 然后 | 但是 | 如果 | 因为 | 而且 | 所以 | 就是 | 其实 | 那么
或者 | 并且 | 还是 | 不过 | 包括 | 比如说 | 另外 | 接下来 | 还有就是 | 就是说
```

**自然结束检测**: 满足以下任一条件：
- 以句号/叹号/问号结尾，且不以连接词结尾
- 匹配特殊自然结束模式：问句（"怎么…呢？"）、总结（"就是这样"）、告别（"拜拜"、"再见"等）

---

### 模块6: 调整分计算

```python
def calculate_adjusted_score(seg) -> float
```

```
调整分 = 基础分
       - (重复次数 / 时长因子) × 5
       - (卡顿次数 / 时长因子) × 3
       - (突然中断 ? 10 : 0)
       + (自然结束 ? 5 : 0)
       + 完整度加分 (0~3, 连续函数, 60秒最佳)
```

其中 `时长因子 = max(1.0, 段落时长 / 30.0)`，用于避免长片段因说话多而累积惩罚。

最终归一化到 0-100。

---

### 模块7: 内容去重 (通用分组策略)

视频内去重和跨视频去重共用同一个底层算法 `_find_duplicate_groups()`：

```python
def _find_duplicate_groups(items, get_text) -> List[Set[int]]
```

1. 两两比较 `get_text(item)` 返回的文本的 `SequenceMatcher` 相似度
2. 相似度 > 阈值 (默认 0.7) 的归为一组

上层函数负责组内选择最优、标记其余为重复：

- `check_duplicate_content(candidates)`: 视频内去重，选择规则 `自然结束 > 调整分 > 段落索引靠后`
- `cross_video_dedup(clips)`: 跨视频去重，选择规则 `自然结束 > 调整分 > 文件名靠后`

---

### 模块8: 分层筛选

```python
def select_best_segment(candidates) -> Segment
```

从排除重复后的候选中，按优先级逐层筛选：

| 层级 | 筛选规则 | 说明 |
|------|---------|------|
| 第1层 | 自然结束优先 | 有自然结束的只保留自然结束的 |
| 第2层 | 流畅度排序 | 按卡顿+重复频率排序，容差1.5次/30秒 |
| 第3层 | 调整分排序 | 保留调整分最高的 |
| 第4层 | 平局处理 | 都没说完→选最后一段；有说完→选最长的 |

---

### 模块9: 剪辑输出

```python
def clip_segment(video_path, seg, output_path) -> bool
```

使用 FFmpeg 剪辑目标段落，前后各加缓冲（默认前1秒后3秒）。

编码参数：H.264 CRF 18（视觉无损）+ AAC 192kbps。

---

### 模块9: FFmpeg 操作

```python
def clip_segment(video_path, seg, output_path) -> bool
def concat_videos(clip_paths, output_path) -> bool
```

- `clip_segment`: 剪辑目标段落，前后加缓冲，H.264 CRF 18 + AAC 192kbps
- `concat_videos`: 使用 FFmpeg concat 协议按列表顺序拼接，重新编码确保格式统一

## API 参考

### 场景A

```python
def process_single_video(video_path: str, output_dir: str, work_dir: str, batch_mode: bool = False) -> Optional[ClipInfo]
```

处理单个视频的完整流程（步骤1-10），返回 `ClipInfo` 供场景B使用，失败返回 `None`。

`batch_mode=True` 时不生成单独报告（由场景B统一生成）。

### 场景B

```python
def process_batch(input_dir: str, output_dir: str, work_dir: str) -> None
```

批量处理流程：
1. 扫描输入目录中的视频文件 (`.MTS`/`.mp4`/`.mov`)，按文件名排序
2. 逐个调用 `process_single_video(batch_mode=True)` 完成粗剪
3. 调用 `cross_video_dedup` 跨视频去重
4. 调用 `concat_videos` 拼接保留的片段
5. **清理中间文件**（单个粗剪视频 + 临时音频）
6. 生成唯一一份批量处理报告（含各片段转录摘要）

最终输出只有两个文件：拼接视频 + 批量报告。

### 主入口

```python
def main() -> None
```

根据第一个参数自动判断场景：
- 参数是目录 → 场景B
- 参数是文件 → 场景A

## 扩展指南

### 添加新的评分维度

1. 在 `Segment` 中添加新字段：

```python
score_content: float = 0  # 内容质量 (20分)
```

2. 在 `score_segment()` 中添加评分逻辑
3. 更新 `total_score` 计算
4. 注意：新增维度后满分会超过100，需同步调整 `min_score` 阈值或做归一化

### 添加新的流畅度检测

1. 在 `analyze_fluency()` 中添加检测逻辑（如语速检测）
2. 在 `calculate_adjusted_score()` 中添加对应的加分/扣分

### 添加新的自然结束模式

在 `analyze_fluency()` 函数的 `special_natural_patterns` 列表中追加正则表达式即可。

### 添加新的视频格式支持

在 `process_batch()` 函数中的 `glob.glob` 调用中添加新的后缀即可（如 `*.avi`）。
